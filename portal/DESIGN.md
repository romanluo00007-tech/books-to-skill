# Portal 设计框架与模块

> 目标：展示所有 skills 和 meta skills，推荐合适的人使用

---

## 一、设计框架选择

### 技术栈建议

| 选项 | 推荐 | 理由 |
|------|------|------|
| React + Vite | ✅ | 与 showcases 一致，可复用组件、设计语言 |
| 纯 HTML/CSS/JS | 备选 | 更轻量，但与 showcases 风格统一成本高 |
| 静态 Markdown | 不推荐 | 交互弱，难以做推荐路径 |

**建议：React + Vite**，和 marketing/strategy showcase 共用同一套视觉体系（深色主题、卡片、导航风格）。

---

### 页面结构

| 方案 | 描述 | 优点 | 缺点 |
|------|------|------|------|
| A. 单页滚动 | 一屏一模块，纵向滚动 | 信息集中、无需跳转 | 内容多时略长 |
| B. 多页导航 | 首页 + 各 skill 子页 | 结构清晰 | 增加跳转、可能过度 |
| C. 单页 + 弹窗 | 主模块在页上，详情用 Modal | 平衡 | 实现稍复杂 |

**建议：方案 A 单页滚动**。Portal 内容量适中（2 个 knowledge skills + 2 个 meta skills，后续扩展销售、产品），单页足够，首屏即可建立全局认知。

---

## 二、模块设计

### 模块一览

```
┌─────────────────────────────────────────────────────────┐
│  1. Hero 主视觉                                          │
├─────────────────────────────────────────────────────────┤
│  2. 我是谁？→ 推荐路径                                    │
├─────────────────────────────────────────────────────────┤
│  3. 知识 Skill 卡片区（营销、战略、销售、产品）             │
├─────────────────────────────────────────────────────────┤
│  4. 元工具 Meta Skill 卡片区                              │
├─────────────────────────────────────────────────────────┤
│  5. 项目结构 / 链接区                                     │
├─────────────────────────────────────────────────────────┤
│  6. Footer                                               │
└─────────────────────────────────────────────────────────┘
```

---

### 模块 1：Hero 主视觉

**目的**：一句话说清「这是什么」「为谁服务」

**内容建议**：
- 主标题：`Books to Skill` 或 `书 → Skill → 展示`
- 副标题：将书籍转化为可复用的 AI 技能
- 简短说明：覆盖营销、战略、销售、产品等方法论，支持一键安装、场景匹配、知识测试

**设计要点**：保持与 showcases 一致的深色背景和简洁排版。

---

### 模块 2：我是谁？→ 推荐路径

**目的**：按角色推荐对应 skill，减少选择成本

**交互形式**：两个身份卡片，点击后高亮推荐内容

| 身份 | 推荐 | 说明 |
|------|------|------|
| **Skill 开发者** | Meta Skills | 把书变成 skill，把 skill 转展示网页 |
| **咨询顾问** | 领域方法论 | 营销、战略、销售、产品等 |

咨询顾问下设子域：营销、战略、销售、产品。每张卡片下方可显示「推荐安装」命令。

---

### 模块 3：知识 Skill 卡片区

**目的**：展示 knowledge skills，引导进入各自 showcase

**当前 + 预留**：
- 营销方法论 ✅（已有 showcase）
- 战略方法论 ✅（已有 showcase）
- 销售方法论 ✅（已有 showcase）
- 产品方法论 ✅（已有 showcase）

**开发中领域**：心理、领导力、个人成长、沟通、思维决策

**每张卡片包含**：
- 领域名称（营销 / 战略 / 销售 / 产品）
- 一句话定位
- 框架数量 + 书籍数量（如有）
- 主色标签（营销金/橙，战略蓝，销售/产品待定）
- 状态：已有 →「进入展示站」「复制安装命令」；开发中 →「敬请期待」

**数据来源**：从各 skill 的 SKILL.md 或已有 showcase 的配置中抽取。销售、产品预留占位卡片，可配置为 `comingSoon: true`。

---

### 模块 4：元工具 Meta Skill 卡片区

**目的**：让「制作者」知道有 books-to-skill、skill-to-showcase 两个元工具

**每张卡片包含**：
- 工具名称
- 一句话说明（输入→输出）
- 适用人群：想生产 skill 的人、想把 skill 做展示的人
- 入口：链接到 `/meta/books-to-skill`、`/meta/skill-to-showcase` 独立详情页

| 元工具 | 输入 | 输出 | 适用人 |
|--------|------|------|--------|
| books-to-skill | 书籍/领域 | 结构化 skill（SKILL.md + references） | 想从书提炼方法论的人 |
| skill-to-showcase | skill 数据 | 可部署的展示网页 | 想给 skill 做落地页的人 |

---

### 模块 5：项目结构 / 链接区

**目的**：说明整体架构，方便开发者或贡献者理解

**内容**：
- 目录树示意图（skills / meta / showcases / portal）
- 各目录一句话说明
- 指向 GitHub 仓库的链接

---

### 模块 6：Footer

**目的**：品牌与开源信息

**内容**：
- 项目名称 + 简短说明
- GitHub 链接
- 可选：OpenClaw / BotLearn 相关链接

---

## 三、设计语言（与 Showcases 统一）

| 要素 | 取值 |
|------|------|
| 背景色 | `#0c0c14` 深色 |
| 卡片背景 | `#16161e`，边框 `#222` |
| 主文字 | `#e5e5e5` |
| 次要文字 | `#888` / `#666` |
| 强调色 | 按模块：营销 `#f0c040`，战略 `#4a90d9`，销售待定，产品待定，Meta `#9b59b6` |
| 圆角 | 12–14px |
| 字体 | `-apple-system, 'Noto Sans SC', sans-serif` |

## 四、实现状态

Portal 已实现：React + Vite、单页滚动、身份推荐、知识 Skill 卡片、Meta Skill 卡片及独立详情页（`/meta/books-to-skill`、`/meta/skill-to-showcase`）。元数据由 `portal/src/config.js` 集中维护。
