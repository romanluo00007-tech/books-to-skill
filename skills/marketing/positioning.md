# 定位理论 (Positioning)
> 来源：《Positioning: The Battle for Your Mind》— Al Ries & Jack Trout（1981）
> 延伸：《22 Immutable Laws of Marketing》— Al Ries & Jack Trout

## 一句话定位

营销的终极战场不是货架，是用户心智。定位就是在用户脑子里占据一个清晰、独特、有价值的位置。

## 核心框架

### 定位的底层逻辑

人的心智有三个特点，定位理论的一切都建立在这之上：
- **心智容量有限**：每个品类，用户最多记住 2-3 个品牌
- **心智厌恶混乱**：模糊的定位等于没有定位
- **心智难以改变**：第一印象一旦形成，扭转成本极高

所以：**你的目标不是让所有人喜欢你，而是在一个具体的品类里成为第一个被想起的名字。**

### 定位四步法

**第 1 步：分析竞争环境**
- 在用户心智中，这个品类里已经有谁？
- 他们各自占据了什么位置？
- 用户提到这个品类时，第一反应是谁？

**第 2 步：找到空位**
- 哪些位置已经被占了？（不要硬抢）
- 哪些位置是空的，但用户在意？
- 空位类型：价格空位（高端/低端）、人群空位（专门服务某类人）、场景空位（专门解决某个问题）、属性空位（独特的功能或特质）

**第 3 步：形成定位声明**
- 模板：「对于 [目标人群]，[品牌名] 是 [品类] 中唯一能 [独特价值] 的，因为 [支撑理由]。」
- 好的定位声明一定是排他的——如果你的竞争对手也能说这句话，那它就不是定位

**第 4 步：贯穿一切触点**
- 定位不是一句 slogan，而是从产品、定价、渠道、内容到视觉的一切决策的过滤器
- 每一个对外动作都要问：这是在强化我的定位，还是在稀释它？

### 关键原则

| 原则 | 说明 |
|------|------|
| 聚焦 | 宁可在一个小品类里称王，也不要在大品类里当路人 |
| 对立 | 如果市场领导者是 A，你就做"非 A"——不要模仿，要对立 |
| 先入 | 第一个进入用户心智比第一个进入市场更重要 |
| 品类 | 如果现有品类没有空位，创造一个新品类 |
| 牺牲 | 定位意味着放弃——你不能什么都是 |

## 怎么用（操作指南）

### 场景 A：为一个品牌/产品做定位

输入：用户告诉你品牌/产品的基本信息、目标市场、竞争对手

执行步骤：
1. 让用户描述或自己搜索 3-5 个主要竞争对手，列出他们各自在用户心智中的印象/标签
2. 画出"心智地图"：横轴和纵轴选两个用户最在意的维度，把竞品放上去
3. 识别空白区域——这些就是潜在的定位机会
4. 对每个空白区域评估：用户真的在意这个位置吗？品牌有能力占据这个位置吗？
5. 选择最有潜力的位置，写出定位声明（用上面的模板）
6. 验证：如果把品牌名换成竞品名，这句话还成立吗？如果成立，说明不够独特，返回第 3 步

### 场景 B：诊断一个品牌为什么"没感觉"

输入：用户觉得自己的品牌/产品"说不清楚是什么""用户记不住"

执行步骤：
1. 问用户：你希望用户提到什么词/场景时想到你？
2. 问用户：你的用户实际上提到什么词/场景时会想到你？
3. 比较 1 和 2 的差距——这就是定位问题的根源
4. 常见病因诊断：
   - **贪心病**：想占多个位置，结果一个都没占住
   - **跟风病**：在已被占据的位置上和领导者硬碰硬
   - **模糊病**：定位描述太抽象，谁都能说
   - **自嗨病**：定位是品牌自己在意的，不是用户在意的
5. 根据病因给出调整建议

### 场景 C：竞品营销分析（定位维度）

输入：用户想了解竞品的营销策略

执行步骤：
1. 收集竞品的对外表达：slogan、官网首页、社交媒体 bio、广告
2. 从这些信息中提取每个竞品试图占据的"心智位置"
3. 画出心智地图
4. 分析：谁的定位最清晰？谁的最模糊？有没有被忽视的空位？
5. 输出竞品定位对比表 + 机会建议

## 判断标准

一个好的定位应该满足：
- [ ] **唯一性**：只有你能这样说，竞品说不了
- [ ] **相关性**：用户真的在意这个差异点
- [ ] **简洁性**：一句话能说清，不需要解释
- [ ] **可持续性**：不会被轻易模仿或超越
- [ ] **可验证**：可以通过具体证据/功能/数据来支撑

---

## 常见误区

1. **把定位等同于 slogan**。定位是战略决策，slogan 只是定位的一种表达方式
2. **以为定位是"我们觉得自己是什么"**。定位是存在于用户心智中的，不是会议室白板上的
3. **定位太宽**。"我们为所有人提供最好的服务"不是定位，是废话
4. **忽视竞争**。定位不是在真空中思考"我是什么"，而是在竞争格局中思考"我和他们有什么不同"
5. **频繁更换定位**。心智建设需要时间，每次换定位都是在归零

## 组合使用

- **定位 + 紫牛 (Purple Cow)**：定位告诉你要占据什么位置，紫牛告诉你怎么让产品本身成为定位的载体
- **定位 + 品牌故事 (StoryBrand)**：定位确定后，用 SB7 框架把定位包装成用户能产生共鸣的故事
- **定位 + SUCCESs**：定位确定后，用 SUCCESs 法则让定位信息更容易被记住
- **定位 + 价值包装 ($100M Offers)**：定位确定方向，Offers 框架设计具体的产品包装和定价

## Agent 指令

当你需要帮用户做定位相关任务时，遵循以下规则：

### 信息收集
在开始分析之前，确保你知道：
- 用户的产品/服务是什么，解决什么问题
- 目标用户是谁（越具体越好）
- 用户知道的竞争对手有哪些
- 用户当前的定位描述（如果有的话）

如果用户没有主动提供，先问。但不要一次问太多问题，分步来。

### 输出格式
根据任务类型输出：

**定位分析**输出：
```
📍 定位分析报告

一、竞争环境
[竞品列表及其心智位置]

二、心智地图
[维度 A] vs [维度 B] 的竞争格局

三、定位机会
[识别出的 2-3 个空白位置及评估]

四、推荐定位
定位声明：「对于……，[品牌] 是……」
选择理由：……

五、落地建议
[3-5 条具体的落地动作]
```

**定位诊断**输出：
```
🔍 定位诊断

当前状态：[描述当前定位的问题]
病因：[贪心 / 跟风 / 模糊 / 自嗨]
建议调整方向：[具体建议]
```

### 注意事项
- 不要直接丢出"心智""品类""空位"这些术语。先用大白话解释，必要时再引入概念
- 永远结合用户的具体行业和产品来讲，不要泛泛而谈
- 如果用户的市场你不熟悉，先问用户补充信息，不要编造竞争格局
- 定位建议要大胆但有据。不要给出四平八稳的"安全"建议——定位的本质就是做取舍
