# 品牌故事框架 SB7
> 来源：《Building a StoryBrand》— Donald Miller（2017）

## 一句话定位

大多数品牌在自说自话。SB7 框架让你停止讲"我们多厉害"，开始讲"我们怎么帮你赢"——把客户变成故事的主角，品牌变成导师。

## 核心框架

### SB7 故事公式

每个好故事（也是每个好品牌叙事）都有 7 个要素：

**1. 主角（Hero）= 你的客户**
- 客户是故事的主角，不是你的品牌
- 主角有一个清晰的欲望/目标
- 关键：用客户的语言描述这个欲望，不要用你的行业术语

**2. 问题（Problem）= 客户面临的挑战**
问题有三层：
- **外部问题**：具体的、可描述的困难（"网站加载太慢"）
- **内部问题**：因外部问题产生的感受（"客户流失让我很焦虑"）
- **哲学问题**：更深层的"这不应该这样"（"好产品不该因为技术问题失败"）
- 大多数品牌只讲外部问题。但用户购买是因为内部问题——他们买的是情感解决方案

**3. 导师（Guide）= 你的品牌**
- 品牌不是主角，是帮助主角的导师（想想尤达之于卢克）
- 导师需要展现两个特质：
  - **共情**：我理解你的困境（"我们知道这有多让人沮丧"）
  - **权威**：我有能力帮你（案例、数据、资质、客户证言）

**4. 计划（Plan）= 明确的行动步骤**
- 客户需要知道"接下来怎么做"——模糊 = 不行动
- 提供清晰的 3-4 步流程（"第一步：预约咨询。第二步：我们制定方案。第三步：你看到效果"）
- 计划的作用是消除不确定性——让客户觉得"这很简单，我能做到"

**5. 行动号召（Call to Action）= 告诉客户该做什么**
- 两种 CTA：
  - **直接 CTA**：明确的购买/注册/预约按钮（"立即购买""免费试用"）
  - **过渡 CTA**：不急于成交，先建立关系（"下载免费指南""参加线上讲座"）
- 大多数品牌的 CTA 太弱或太模糊。"了解更多"不是 CTA，是推诿

**6. 避免失败（Failure）= 不行动的代价**
- 人对"失去"比对"获得"更敏感（损失厌恶）
- 要温和但清晰地展示：如果不解决这个问题，会怎样？
- 不是恐吓，而是帮客户认清风险："如果继续这样，6 个月后可能……"

**7. 成功结局（Success）= 行动后的美好图景**
- 具体描述客户成功后的状态——让他们"看到"自己想要的未来
- 三种成功方式：
  - **获得地位/权力**："成为行业标杆"
  - **完成联结**："团队终于同频"
  - **实现自我**："做自己真正想做的事"
- 避免模糊的承诺，要具体："从每周加班 20 小时变成准时下班"

## 怎么用（操作指南）

### 场景 A：为品牌/产品写官网文案

输入：用户有一个产品/服务，需要写出能转化的官网文案

执行步骤：
1. 用 SB7 七要素逐个提问，收集信息：
   - 你的客户是谁？他们想要什么？
   - 他们面临什么问题（外部/内部/哲学层面）？
   - 你怎么帮助他们？（共情+权威）
   - 合作流程是什么？（3-4 步）
   - 你希望他们做什么？（CTA）
   - 不行动会怎样？
   - 行动后会变成什么样？
2. 用收集到的信息填充 SB7 模板
3. 按"首屏—问题展开—导师介绍—计划—CTA—成功愿景"的结构输出文案
4. 检查：主语是"你"还是"我们"——好的品牌文案里"你"的出现频率应该远高于"我们"

### 场景 B：优化现有品牌表达

输入：用户有现成的品牌文案/官网/介绍，觉得"不够打动人"

执行步骤：
1. 用 SB7 框架审查现有内容，逐个要素检查是否到位
2. 最常见问题：
   - 品牌把自己当主角（通篇在讲"我们的技术""我们的团队"）
   - 只讲外部问题，没有触及内部问题
   - 没有明确的行动步骤
   - CTA 模糊或缺失
3. 给出具体的改写建议，标注每处修改对应 SB7 的哪个要素

### 场景 C：品牌 Campaign / 单次传播

输入：用户要做一次营销活动/社交媒体传播

执行步骤：
1. 用 SB7 精简版构建 Campaign 叙事：
   - 你的客户在面对什么挑战？（Problem，制造共鸣）
   - 我们能怎么帮？（Guide，一句话价值主张）
   - 该怎么做？（Plan + CTA，明确下一步）
2. 输出 Campaign 核心信息 + 各触点的内容建议

## 判断标准

一个好的品牌叙事应该满足：
- [ ] **客户是主角**，不是品牌。检查：通篇"你"多还是"我们"多？
- [ ] **问题触及内部感受**，不只是功能描述
- [ ] **品牌展现了共情+权威**，而不是自吹自擂
- [ ] **有清晰的 3-4 步计划**，客户知道接下来做什么
- [ ] **CTA 明确具体**，不是"了解更多"
- [ ] **既展示了不行动的风险，也展示了行动后的美好图景**

---

## 常见误区

1. **品牌自恋**。把自己当成故事的主角。"我们成立于2010年，拥有100项专利……"——用户不关心，他们关心的是"你能帮我解决什么"
2. **只有外部问题**。只讲功能层面的问题和解决方案，没有触及用户的情感和深层需求
3. **导师形象过于强势**。导师不是英雄——不要变成"我们无所不能"，而是"我们理解你，我们能帮你"
4. **没有计划**。用户看完觉得"好像不错"但不知道接下来干嘛，就流失了
5. **不敢讲失败**。怕吓走客户所以避而不谈风险，但没有风险就没有行动的紧迫性

## 组合使用

- **SB7 + 定位理论**：定位确定品牌该占据什么心智位置，SB7 把这个定位转化成客户能产生共鸣的故事
- **SB7 + STEPPS**：SB7 构建故事本体，STEPPS 确保故事有传播力（尤其是 Social Currency 和 Emotion 维度）
- **SB7 + 价值包装 ($100M Offers)**：SB7 讲"为什么选你"，Offers 讲"凭什么这个价格值"

## Agent 指令

### 信息收集
按 SB7 的七要素逐步收集。不需要一次问完，按对话自然推进：
1. 先了解产品/服务是什么 → 自然引出客户和问题
2. 再了解品牌的优势 → 对应导师角色
3. 最后确认流程和 CTA → 对应计划和行动

### 输出格式

**品牌叙事**输出：
```
📖 品牌故事框架（SB7）

主角：[目标客户画像，一句话]
欲望：[客户想要什么]

问题：
  外部：[具体困难]
  内部：[因此产生的感受]
  哲学：[这不应该这样]

导师：[品牌名] 
  共情：[我理解你……]
  权威：[我能帮你因为……]

计划：
  1. [第一步]
  2. [第二步]  
  3. [第三步]

行动号召：[具体 CTA]

如果不行动：[风险/代价]
行动后的成功：[具体的美好图景]

---
基于以上框架的文案：
[输出实际可用的文案]
```

### 注意事项
- SB7 最强的应用是"官网/着陆页文案"和"品牌介绍"。当用户说"帮我写官网""帮我介绍我的产品"时，默认用 SB7
- 最核心的一个翻转：帮用户意识到"你不是故事的英雄，你的客户才是"。很多用户会本能地以品牌视角来讲故事，要温和但坚定地引导他们切换视角
- 写出来的文案必须"说人话"——避免行业黑话和企业腔
