# 价值包装 ($100M Offers)
> 来源：《$100M Offers: How to Make Offers So Good People Feel Stupid Saying No》— Alex Hormozi（2021）

## 一句话定位

不是降价才能卖出去。Hormozi 的框架教你把产品包装成"不买就亏了"的报价——通过提升感知价值而不是降低价格。

## 核心框架

### 价值方程式

```
价值 = (理想结果 × 感知实现概率) ÷ (时间延迟 × 努力与牺牲)
```

要提升价值，你有四个杠杆：
- **理想结果** ↑ ：让结果更大、更吸引人
- **感知实现概率** ↑ ：让客户觉得"这真的能成"
- **时间延迟** ↓ ：让客户更快看到效果
- **努力与牺牲** ↓ ：让客户更省事

大多数人只会动"降价"这一个杠杆。Hormozi 的核心观点是：**价格不是问题，价值才是。**

### 构建不可抗拒报价的步骤

**第 1 步：识别理想结果（Dream Outcome）**
- 客户真正想要的是什么？不是你的产品功能，是那个功能带来的最终状态
- "客户不是要买钻头，是要墙上那个洞"——再往前一步："客户要的不是洞，是挂上全家福照片后的温馨客厅"
- 把结果描述得越具体、越有画面感越好

**第 2 步：列出所有障碍（Problems）**
- 在客户实现理想结果的路上，有哪些障碍？
- 不只是购买前的障碍，还有购买后的障碍（用不会、没时间、怕失败……）
- 列出越多越好——每个障碍都是一个你可以解决的价值点

**第 3 步：为每个障碍设计解决方案（Solutions）**
- 每个障碍对应一个解决方案，每个解决方案就是你报价中的一个组件
- 解决方案可以是：产品功能、附加服务、工具、模板、社群、1对1辅导、保证承诺
- 关键：不要只想"我能做什么"，要想"什么能让客户的障碍消失"

**第 4 步：组合成报价堆叠（Offer Stack）**
- 把所有解决方案打包成一个看起来价值远超价格的报价
- 结构：核心产品 + 增值组件 + 附赠/奖励 + 保证
- 每个组件都标注"独立价值"，最后加总 vs 实际售价——让差距触目惊心

**第 5 步：增强器**
- **稀缺性**：限量（真实的）
- **紧迫感**：限时（真实的）
- **保证**：降低风险（退款保证、效果保证、按结果付费）
- **命名**：给你的报价起一个有吸引力的名字，不要叫"标准套餐"

### 定价原则

- **不要做价格最低的**——低价吸引最差的客户，高价吸引最好的客户
- **定价依据是价值，不是成本**——如果你能帮客户赚 100 万，收 10 万不贵
- **先涨价，再提供更多价值来匹配**——而不是先降价来吸引客户

## 怎么用（操作指南）

### 场景 A：设计产品/服务的报价方案

输入：用户有产品/服务，想设计一个让人无法拒绝的报价

执行步骤：
1. 明确客户的理想结果——用客户的语言描述，不用产品术语
2. 列出实现理想结果路上的所有障碍（至少 10 个，越多越好）
3. 为每个障碍设计解决方案
4. 按"高价值、低成本"排序——优先选那些对客户价值高但你交付成本低的方案
5. 组合成报价堆叠：核心产品 + 3-5 个增值组件 + 保证
6. 为每个组件估算"如果单独购买的价值"
7. 设定一个远低于总价值的实际价格
8. 给报价起一个有吸引力的名字
9. 输出完整的报价方案

### 场景 B：诊断"为什么卖不动"

输入：用户的产品有人看但转化低

执行步骤：
1. 用价值方程式做诊断：
   - 理想结果清楚吗？有吸引力吗？
   - 客户觉得能实现吗？（信任问题）
   - 多久能看到效果？（太慢会犹豫）
   - 客户需要付出多少努力？（太麻烦会放弃）
2. 找到最弱的环节——通常要么是"结果不够吸引"要么是"风险太高"
3. 针对弱项提出改进方案
4. 检查是否有保证机制——没有保证是转化低的最常见原因之一

### 场景 C：产品定价——"我该卖多少钱"

输入：用户不知道怎么定价

执行步骤：
1. 先搞清楚客户获得的价值——你的产品帮客户赚多少钱/省多少钱/避免多少损失？
2. 定价参考：价值的 1/10 到 1/5 是常见的合理区间
3. 用报价堆叠拉开感知价值和实际价格的差距
4. 如果客户觉得贵，问题不在价格——回到价值方程式找问题
5. 建议：宁可定高价后提供超预期服务，也不要定低价然后发现利润不够提供好服务

## 判断标准

一个不可抗拒的报价应该：
- [ ] **理想结果具体且有吸引力**——客户一看就想要
- [ ] **报价堆叠的总价值至少是实际价格的 3-10 倍**
- [ ] **有风险消除机制**（退款保证/效果保证/免费试用）
- [ ] **每个组件都解决一个真实的客户障碍**，不是凑数的赠品
- [ ] **有独特的命名**，不叫"基础版/高级版"
- [ ] **有稀缺性或紧迫感**——但必须真实

---

## 常见误区

1. **赠品堆砌**。给一堆跟核心产品无关的东西来"凑价值"——用户看得出来。每个组件必须解决一个真实障碍
2. **不敢定高价**。低价吸引来的客户往往最难服务、最容易投诉。价格筛选客户质量
3. **没有保证**。"如果不满意怎么办？"——如果你不回答这个问题，客户会自己回答：不买
4. **只描述功能不描述结果**。"包含12节视频课程"不如"30天内掌握XX技能"
5. **价值和价格差距不够大**。如果你的总价值是 5000，卖 4000——这不是不可抗拒。要让差距大到客户觉得"不买就亏了"

---

## 组合使用

- **Offers + 定位理论**：定位确定"你是什么"，Offers 设计"你卖什么、怎么卖"。定位在先，报价在后
- **Offers + 影响力原则 (Cialdini)**：Offers 中的稀缺性、保证都是影响力原则的具体应用。两者配合可以精确设计转化路径
- **Offers + 品牌故事 (StoryBrand)**：StoryBrand 解决"为什么选你"（情感层面），Offers 解决"值不值"（理性层面）
- **Offers + 紫牛**：如果产品本身足够"紫牛"，Offers 的说服难度会大幅降低

## Agent 指令

### 信息收集
- 用户的产品/服务具体是什么
- 价格（或计划定价）
- 客户是谁，客户的理想结果是什么
- 当前的报价结构（如果已有）
- 竞品的价格区间

### 输出格式

**报价设计**输出：
```
💰 不可抗拒的报价方案

客户的理想结果：[描述]

障碍 → 解决方案：
1. [障碍] → [解决方案]（独立价值：¥XX）
2. [障碍] → [解决方案]（独立价值：¥XX）
3. ……

报价堆叠：
🎯 核心：[核心产品/服务]
➕ 增值1：[组件]（价值 ¥XX）
➕ 增值2：[组件]（价值 ¥XX）
➕ 增值3：[组件]（价值 ¥XX）
🎁 额外赠送：[如果快速行动]
🛡️ 保证：[风险消除机制]

总价值：¥XX,XXX
你的投资：¥X,XXX

报价名称建议：[一个有吸引力的名字]
```

### 注意事项
- 这是最直接帮用户赚钱的框架。用户做产品包装、定价咨询时，优先用这个
- "独立价值"估算要合理——不是随便编一个大数字，要有依据（市场上类似服务的价格、客户为此付出的替代成本等）
- 强调保证的重要性——很多用户害怕给保证，要帮他们理解：好的保证筛选好客户，只有对产品没信心才不敢给保证
- 如果用户的产品本身有问题（质量不行、需求不存在），不要只帮他"包装"——要诚实指出根本问题
