# STEPPS 传播模型
> 来源：《Contagious: Why Things Catch On》— Jonah Berger（2013）

## 一句话定位

内容的传播不是随机的。STEPPS 是 6 个让内容自带传播力的引擎——每多命中一个，传播概率就多一层。

## 核心框架

### 六个传播引擎

**S — Social Currency（社交货币）**
人们分享让自己显得聪明/有趣/有品味/内行的内容。
- 关键问题：分享这个内容，能让分享者在社交圈里"加分"吗？
- 实操手段：独家信息、反常识洞察、测试/排名、稀缺限定、内幕消息

**T — Triggers（触发器）**
容易被日常场景提醒的内容，被谈论的频率更高。
- 关键问题：用户日常生活中有没有一个高频场景会"触发"他们想起这个内容？
- 实操手段：绑定高频场景（天气、节日、通勤、午餐）、绑定常见情绪、绑定日常物品
- 经典案例：KitKat 绑定"咖啡时间"，每次喝咖啡就想到 KitKat

**E — Emotion（情绪）**
高唤醒情绪驱动分享，低唤醒情绪抑制分享。
- 高唤醒（促进分享）：敬畏、兴奋、愤怒、焦虑、幽默
- 低唤醒（抑制分享）：满足、悲伤、平静
- 关键问题：这个内容看完，用户是想"转发给朋友看"还是想"嗯，挺好的"然后划走？
- 注意：正面情绪和负面情绪都能驱动分享，关键是唤醒度，不是正负

**P — Public（公共可见性）**
能被看到的行为会被模仿。越容易观察到别人在用/在看，越容易传播。
- 关键问题：用户使用/分享你的内容/产品时，别人能看到吗？
- 实操手段：外显的品牌标识、公开的使用痕迹、排行榜/热门榜单、"已有 XX 人参与"
- 经典案例：苹果"Sent from my iPhone"签名，把私人行为变成公共可见

**P — Practical Value（实用价值）**
人们分享有用的东西，帮助他人也是一种社交货币。
- 关键问题：看完这个内容，用户能不能立刻用来解决一个具体问题？
- 实操手段：省钱技巧、操作教程、避坑指南、效率工具推荐、清单/模板
- 注意：实用信息需要包装——不是信息越多越好，而是要"令人惊讶地有用"

**S — Stories（故事）**
信息需要搭载在故事上才能传播。人不转发说明书，但会转发故事。
- 关键问题：你的核心信息是嵌在一个故事里的，还是赤裸裸地呈现的？
- 关键：好的传播故事有"特洛伊木马"效应——人们是为了故事本身去分享，但你的信息搭了故事的便车
- 注意：故事必须和品牌/信息不可分割。如果把品牌名拿掉故事依然成立，那这个故事对你没用

## 怎么用（操作指南）

### 场景 A：分析一条内容为什么火了

输入：用户给出一条爆款内容（小红书帖子、抖音视频、推文、文章等）

执行步骤：
1. 逐个维度扫描这条内容，判断它命中了 STEPPS 中的哪几个
2. 对每个命中的维度，指出具体是怎么命中的
3. 找出它最强的 1-2 个传播引擎——这是它火的主因
4. 分析它没有命中的维度——这是潜在的加强空间
5. 总结：一句话概括"这条内容火的核心原因"

### 场景 B：分析一条内容为什么没火

输入：用户给出自己发的内容，效果不好

执行步骤：
1. 用 STEPPS 六维度扫描，标记每个维度的得分（强/中/弱/无）
2. 找到最薄弱的维度——这通常是"不火"的主因
3. 对薄弱维度给出具体改进建议
4. 如果整体都弱，建议用户先聚焦在最容易提升的 1-2 个维度
5. 提供改进版的思路（不是直接重写，而是告诉用户怎么调整方向）

### 场景 C：帮用户策划有传播力的内容

输入：用户要发布一条内容，想让它有更好的传播效果

执行步骤：
1. 了解用户的核心信息/目标、目标受众、发布平台
2. 评估这个内容主题天然适合哪几个 STEPPS 维度
3. 选择 2-3 个最适合发力的维度（不需要六个都强）
4. 为每个选中的维度提供具体的操作建议：
   - 社交货币：怎么让受众觉得"分享这个显得我很XX"
   - 触发器：绑定什么日常场景
   - 情绪：打什么情绪点，用什么手法
   - 公共性：怎么让分享行为可见
   - 实用价值：怎么让它"有用到忍不住转"
   - 故事：用什么叙事结构
5. 输出内容策略建议（不是直接写内容，而是内容的"设计图纸"）

### 场景 D：批量内容策划/内容日历

输入：用户需要规划一段时间的内容

执行步骤：
1. 在不同时间段轮换侧重不同的 STEPPS 维度，避免单一化
2. 为每条内容标注主打的传播引擎
3. 确保整体内容矩阵在六个维度上有覆盖

## 判断标准

一条有传播力的内容至少应该：
- [ ] 在 STEPPS 六维度中**强命中至少 2 个**
- [ ] 有一个**清晰的情绪锚点**（看完后的第一情绪反应是什么？）
- [ ] 分享后对分享者的**社交形象有加分**
- [ ] 核心信息**不脱离故事独立存在**（即不能把故事去掉只剩干巴巴的信息）

## 常见误区

1. **以为好内容自然会传播**。好内容只是前提，传播需要设计。一条信息量很大但没有情绪钩子的内容，可能"有用"但不会被分享
2. **只看情绪，忽视社交货币**。"好感动"不一定导致分享。要问的是"看完之后会不会转发给朋友"，而不只是"看完有没有感觉"
3. **把六个维度当 checklist 去凑**。不需要每条内容都命中六个维度，强命中 2-3 个比弱命中 6 个有效得多
4. **混淆高唤醒和负面情绪**。悲伤是负面的低唤醒情绪，不利于传播；愤怒是负面的高唤醒情绪，反而促进传播。关键是唤醒度
5. **故事和品牌两张皮**。如果用户转发的时候会自然地略去你的品牌名，说明故事和品牌没有绑定成功

## 组合使用

- **STEPPS + SUCCESs**：STEPPS 解决"让人分享"，SUCCESs 解决"让人记住"。先用 SUCCESs 让信息本身有粘性，再用 STEPPS 设计传播策略。两者配合是内容创作的最强组合
- **STEPPS + 品牌故事 (StoryBrand)**：STEPPS 中的"S(Stories)"维度可以直接用 SB7 框架来设计故事结构
- **STEPPS + 六大影响力 (Cialdini)**：影响力原则可以强化 STEPPS 中的多个维度——稀缺性增强社交货币，社会认同增强公共可见性，互惠增强实用价值

## Agent 指令

### 信息收集
在开始分析之前，确保你知道：
- 要分析/创作的内容是什么（具体内容或主题方向）
- 目标受众是谁（他们在意什么、社交圈是什么样的）
- 发布平台（不同平台的传播机制不同：小红书靠搜索+推荐、抖音靠算法推荐、微信靠社交转发、Twitter靠引用转发）
- 用户的目标是传播量（曝光）还是转化（行动）——这会影响维度侧重

### 输出格式

**爆款分析**输出：
```
🔥 STEPPS 传播分析

内容概述：[一句话描述]

传播引擎扫描：
✅ Social Currency（社交货币）：[强/中/弱] — [具体说明]
✅ Triggers（触发器）：[强/中/弱] — [具体说明]
✅ Emotion（情绪）：[强/中/弱] — [情绪类型+唤醒度]
✅ Public（公共可见性）：[强/中/弱] — [具体说明]
✅ Practical Value（实用价值）：[强/中/弱] — [具体说明]
✅ Stories（故事）：[强/中/弱] — [具体说明]

核心传播引擎：[最强的 1-2 个维度]
火的原因（一句话）：[总结]

优化空间：[可以加强的维度和具体建议]
```

**内容策划**输出：
```
📋 内容传播策略

核心信息：[用户要传达的]
目标受众：[谁]
主攻维度：[选中的 2-3 个 STEPPS 维度]

策略设计：
[维度1]：[具体怎么做]
[维度2]：[具体怎么做]
[维度3]：[具体怎么做]

内容方向建议：[基于以上策略的 2-3 个内容角度]
```

### 注意事项
- 分析爆款时要具体。不要说"这条内容有社交货币"，而要说"分享这条内容会让分享者在朋友圈显得懂行/有品味/消息灵通，因为……"
- 不同平台的传播逻辑不同，要结合平台特性来分析。小红书强实用价值、抖音强情绪和故事、微信公众号强社交货币、B站强内容深度
- 帮用户策划内容时，先给策略方向，不要直接写成品——让用户基于策略自己判断再决定是否需要你生成具体内容
- 如果用户给的是一条不太好的内容让你"改好"，不要只修修补补。先做 STEPPS 诊断，找到根本问题，再建议是微调还是换方向
