# 上瘾模型 (Hook Model)
> 来源：《Hooked: How to Build Habit-Forming Products》— Nir Eyal（2014）

## 一句话定位

用户用一次就走了？上瘾模型是一个四步循环，解释为什么有些产品让人"放不下"——以及怎么设计这种粘性。

## 核心框架

### Hook 循环四步

```
触发(Trigger) → 行动(Action) → 奖赏(Reward) → 投入(Investment)
      ↑                                                    |
      └────────────────────────────────────────────────────┘
```

**1. 触发（Trigger）— 什么让用户"打开"你的产品**

两种触发：
- **外部触发**：推送通知、邮件、广告、朋友推荐、看到别人在用
- **内部触发**：用户内心的情绪或需求——无聊、焦虑、孤独、好奇、FOMO
- 目标：从外部触发起步，最终建立内部触发。当用户一感到[某种情绪]就自动想到你的产品，Hook 就建立了
- 关键问题：用户在什么情绪/场景下会想到用你的产品？

**2. 行动（Action）— 用户做的最小动作**

行为 = 动机 + 能力 + 触发（Fogg 行为模型）
- **动机**要够：用户想要这个结果
- **能力**要高：动作要尽可能简单（滑动、点击、输入一个词）
- 简化路径：减少步骤、减少思考、减少等待
- 关键问题：从触发到获得奖赏，用户需要几步？每一步能不能更简单？

**3. 奖赏（Variable Reward）— 让用户觉得"值了"**

关键词是"可变"——可预测的奖赏让人满足，不可预测的奖赏让人上瘾。

三种奖赏类型：
- **社交奖赏（Tribe）**：来自他人的认可——点赞、评论、关注增长
- **猎物奖赏（Hunt）**：获取资源/信息的快感——刷到好内容、找到好deal、开箱
- **自我奖赏（Self）**：个人成就感——完成任务、升级、技能提升

可变性是核心：不知道这次刷会看到什么 → 持续刷。不知道这个通知是什么 → 立刻点开

**4. 投入（Investment）— 让用户"沉没"进来**

用户在产品中投入的时间、数据、内容、社交关系、学习成本——这些都让离开变得更难。
- 投入 → 产品变得更好（个性化推荐、积累的内容、建立的关系）
- 投入 → 增加下一次触发的可能（发布内容后等待别人回应 → 外部触发回来）
- 关键：投入不是当下的消费，是为未来价值做储蓄

## 怎么用（操作指南）

### 场景 A：诊断产品留存问题——"用户用一次就走了"

输入：用户的产品有新用户但留存差

执行步骤：
1. 用 Hook 四步逐一检查：
   - 触发：用户第一次用完后，有什么机制让他们回来？内部触发建立了吗？
   - 行动：核心操作是否足够简单？首次使用体验有没有摩擦？
   - 奖赏：用户每次使用能获得什么？奖赏有变化性吗？
   - 投入：用户有没有在产品里留下东西，让离开有成本？
2. 找到循环断裂的环节——通常是"没有内部触发"或"奖赏没有可变性"
3. 为断裂环节设计修复方案

### 场景 B：为新产品设计粘性机制

输入：用户在设计产品，想从一开始就考虑留存

执行步骤：
1. 确定核心内部触发——用户在什么情绪/时刻会想到用这个产品？
2. 设计最简化的核心行动——从打开到获得价值的路径
3. 设计可变奖赏——这个产品能给用户哪种类型的意外满足？
4. 设计投入机制——用户怎么在产品中积累价值？
5. 画出完整的 Hook 循环图
6. 检查循环是否闭合——投入是否能增强下一次的触发？

### 场景 C：分析竞品为什么让人"上瘾"

输入：用户想理解某个竞品/产品为什么用户粘性高

执行步骤：
1. 拆解它的 Hook 循环——触发、行动、奖赏、投入分别是什么
2. 找到它最强的环节
3. 分析可借鉴的机制
4. 注意：不是所有"上瘾"机制都适合照搬，要考虑产品类型和用户群

## 判断标准

一个有粘性的产品循环应该：
- [ ] **有明确的内部触发**——某种情绪/场景会让用户想到你
- [ ] **核心行动极其简单**——几秒内完成
- [ ] **奖赏有可变性**——每次使用的体验不完全相同
- [ ] **用户有投入**——使用越多，积累越多，越不想走
- [ ] **循环闭合**——投入阶段自然导向下一次触发

## 常见误区

1. **以为上瘾=游戏化**。加积分、勋章、排行榜只是表面。如果核心产品没有价值，游戏化只是在无聊上面加糖衣
2. **忽视内部触发**。只靠推送和邮件（外部触发）拉人回来，没有建立内部触发——一旦用户关闭通知就失联了
3. **奖赏太可预测**。每次打开都是同样的体验，没有惊喜感。可变奖赏是习惯形成的关键
4. **不考虑伦理**。上瘾模型是强力工具，需要负责任地使用。帮用户设计的应该是"有价值的习惯"，不是利用心理弱点的成瘾机制
5. **投入环节缺失**。很多产品用完即走——没有让用户留下数据、内容、关系，下次使用就要从零开始

## 组合使用

- **Hook + STEPPS**：Hook 解决"让老用户留下来"，STEPPS 解决"让老用户带新用户来"。留存+传播双引擎
- **Hook + 影响力原则**：承诺一致性原则可以强化投入环节，社会认同可以强化社交奖赏
- **Hook + 价值包装 ($100M Offers)**：Offers 解决首次转化，Hook 解决转化后的留存。先让人买，再让人留

## Agent 指令

### 信息收集
- 产品是什么类型（工具、社交、内容、电商等）
- 用户使用频率预期（日活？周活？月活？）
- 当前的留存数据（如果有）
- 用户在什么场景下使用产品

### 输出格式

**留存诊断**输出：
```
🔄 Hook 循环诊断

触发：[现状评估] — [是否建立了内部触发]
行动：[现状评估] — [核心操作是否足够简单]
奖赏：[现状评估] — [是否有可变性]
投入：[现状评估] — [用户是否在积累价值]

循环闭合性：[闭合 / 断裂于X环节]

优化建议：
1. [最优先修复的环节 + 具体方案]
2. [次优先]
3. [补充]
```

### 注意事项
- 这个框架最适合产品经理和创业者——当用户的问题是"留存""粘性""用户习惯"时用它
- 如果用户做的是低频产品（比如婚庆、装修），Hook 模型可能不完全适用——要调整预期
- 要有伦理意识：帮用户设计有价值的习惯循环，如果用户的产品本质上是利用成瘾性获利（赌博类、无节制消费类），应该指出这一点
