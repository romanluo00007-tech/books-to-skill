# 蓝海战略 (Blue Ocean Strategy)
> 来源：《Blue Ocean Strategy》— W. Chan Kim & Renée Mauborgne（2005）
> 延伸：《Blue Ocean Shift》— W. Chan Kim & Renée Mauborgne（2017）

## 一句话定位

不在现有市场里跟对手死磕（红海），而是重新定义竞争边界、创造没人争的新市场空间（蓝海）。适合想跳出价格战、寻找增长新曲线的企业。

## 核心框架

### 战略画布 (Strategy Canvas)

一张图看清行业竞争的现状：

- **横轴**：行业竞争的关键要素（行业里大家比什么）
- **纵轴**：每个要素的投入/表现水平（高-中-低）
- **价值曲线**：把你和竞争对手在每个要素上的表现连成线

用途：看清"行业里所有人都在用差不多的方式竞争"——如果大家的价值曲线形状差不多，说明在红海里。

### 四步动作框架 (ERRC Grid)

跳出红海的四个动作：

| 动作 | 核心问题 | 目的 |
|------|---------|------|
| **消除 (Eliminate)** | 行业里哪些"理所当然"的要素其实可以去掉？ | 降成本、简化 |
| **减少 (Reduce)** | 哪些要素可以降到行业标准以下？ | 减少过度竞争的浪费 |
| **增加 (Raise)** | 哪些要素应该提到行业标准以上？ | 解决行业长期忽视的痛点 |
| **创造 (Create)** | 行业从来没有提供过的、哪些全新要素应该被创造？ | 开辟新需求 |

**关键原则**：
- 四个动作要一起用，不是只做"增加"和"创造"（那只是加功能加成本）
- "消除"和"减少"同样重要——它们降低成本，让你能在新要素上投入更多
- 好的蓝海战略同时实现差异化和低成本，而不是二选一

### 价值创新 (Value Innovation)

蓝海战略的核心理念：
- 不在现有竞争维度上做得"比别人好一点"
- 而是重新定义竞争维度，让竞争变得不相关
- 价值创新 = 同时为买方创造更高价值 + 降低企业成本

### 三层非客户分析

找蓝海不是盯着现有客户，而是看三层"非客户"：

| 层级 | 定义 | 特征 |
|------|------|------|
| **第一层** | 行业边缘用户 | 勉强在用，随时可能离开 |
| **第二层** | 拒绝型非客户 | 看过但主动选择不用 |
| **第三层** | 未探索的非客户 | 从来没被当作目标客户 |

## 怎么用（操作指南）

### 场景 A：为现有业务寻找蓝海机会

输入：用户有一个现有业务，竞争激烈，想找到差异化突破口

执行步骤：
1. 画出行业战略画布——列出行业竞争的 6-8 个关键要素
2. 画出自己和主要竞争对手的价值曲线
3. 观察：大家的曲线是否高度相似？如果是，说明处于红海
4. 用四步动作框架逐个问：哪些要素可以消除？减少？增加？创造？
5. 画出新的价值曲线——确保形状和行业现有曲线明显不同
6. 用三层非客户分析验证：新曲线能吸引哪些现在不是你客户的人？
7. 输出蓝海战略方案

### 场景 B：评估一个新业务/产品创意是否是"真蓝海"

输入：用户有一个新想法，想判断是不是真正的蓝海

执行步骤：
1. 画出该行业的战略画布
2. 把用户的创意放上去，看价值曲线是否与行业明显不同
3. 检查四步动作：创意中是否同时有"消除/减少"和"增加/创造"？
4. 如果只有"增加/创造"没有"消除/减少"——不是蓝海，是高成本差异化
5. 检查是否开拓了非客户——蓝海不是从对手那里抢人，是扩大市场
6. 给出判断：真蓝海 / 伪蓝海 / 需要调整的方向

### 场景 C：用蓝海思维做产品/服务重新设计

输入：用户想重新设计产品或服务线

执行步骤：
1. 列出当前产品/服务的所有功能和特性
2. 按四步动作框架分类：哪些该消除、减少、增加、创造
3. 特别关注"行业默认提供但客户其实不在意"的东西——这是消除的最佳候选
4. 用非客户视角检验：重新设计后的产品能吸引谁？
5. 估算成本变化：消除和减少的部分是否足以支撑增加和创造的投入？
6. 输出重新设计方案和新旧对比

## 判断标准

- [ ] **价值曲线独特** — 新策略的价值曲线和行业现有曲线明显不同，不是微调
- [ ] **四步动作齐全** — 不只是加东西，也在减东西。消除/减少和增加/创造并存
- [ ] **同时降成本+提价值** — 不是花更多钱做差异化，而是通过取舍实现双赢
- [ ] **非客户视角** — 策略能吸引现有行业之外的人，不只是抢现有客户
- [ ] **竞争变得不相关** — 不是在同一维度上做得"比对手好"，而是换了维度

## 常见误区

1. **把蓝海等于创新**。蓝海不是技术创新或发明新东西，而是价值组合的创新。很多蓝海用的都是成熟技术，只是重新组合。
2. **只做加法不做减法**。只想着"增加"和"创造"新功能，不愿意"消除"和"减少"——结果成本飙升，不是蓝海而是烧钱。
3. **盯着现有客户找蓝海**。蓝海的机会往往在非客户身上。如果只调研现有客户，你得到的只是"改良版红海"。
4. **以为蓝海能永远蓝**。蓝海会被模仿变成红海，要做好持续创新的准备。关键是建立模仿壁垒。
5. **战略画布画了但不用**。画完战略画布是为了发现洞察，不是为了做PPT。如果画完之后还是老打法，那就是浪费时间。

## 组合使用

- **蓝海战略 + 竞争战略**：先用 Porter 五力看清红海有多惨烈，如果五力都很强，就用蓝海四步动作框架跳出去。Porter 是"诊断"，蓝海是"开处方"。
- **蓝海战略 + 从0到1**：蓝海的"创造新市场"和 Thiel 的"从0到1"理念高度一致。蓝海提供方法论工具（战略画布、ERRC），Zero to One 提供思维哲学（垄断思维、逆向思考）。
- **蓝海战略 + 颠覆式创新**：蓝海关注"重新定义价值"，颠覆式创新关注"从低端或新市场切入"。两者可以叠加——用颠覆式创新选进入点，用蓝海重新定义价值组合。

## Agent 指令

### 信息收集
在执行前需要知道：
- 用户在哪个行业/市场？（越具体越好）
- 行业里主要的竞争要素有哪些？（大家比什么？价格？品质？速度？服务？）
- 主要竞争对手是谁，他们各自的特点？
- 用户目前的业务状态——已有业务想突破，还是在规划新业务？
- 有没有注意到一些"不是你的客户但理论上可以是"的群体？

先问行业和目的，其他在分析过程中逐步挖掘。

### 输出格式

**蓝海机会分析**输出：
```
🌊 [行业名] 蓝海机会分析

📊 战略画布：
[列出 6-8 个竞争要素，标注行业平均水平和主要对手水平]

🔄 四步动作方案：
❌ 消除：[要素列表 + 理由]
⬇️ 减少：[要素列表 + 理由]
⬆️ 增加：[要素列表 + 理由]
✨ 创造：[要素列表 + 理由]

🎯 新价值曲线：[描述新策略的核心差异]
👥 目标非客户：[三层非客户中最大的机会]
💰 成本影响：[消除/减少节省的 vs 增加/创造投入的]

⚠️ 风险与壁垒：[被模仿的风险 + 建议的防御手段]
```

**蓝海验证**输出：
```
🔍 [创意名] 蓝海验证

✅/❌ 价值曲线独特性：[判断 + 理由]
✅/❌ 四步动作完整性：[判断 + 理由]
✅/❌ 成本-价值双赢：[判断 + 理由]
✅/❌ 非客户吸引力：[判断 + 理由]

📊 综合判断：[真蓝海 / 伪蓝海 / 有潜力但需调整]
💡 调整建议：[如果不是真蓝海，怎么改]
```

### 注意事项
- 战略画布要尽量用具体的、可量化的竞争要素，不要用模糊的"品质""服务"
- 四步动作框架中，"消除"往往最有价值也最难——行业习以为常的东西最难被质疑
- 蓝海分析不能纸上谈兵，要鼓励用户去和"非客户"聊聊验证
- 如果行业信息不足，用搜索补充行业数据后再做分析
