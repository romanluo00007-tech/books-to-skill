# 好战略坏战略 (Good Strategy Bad Strategy)
> 来源：《Good Strategy Bad Strategy》— Richard Rumelt（2011）

## 一句话定位

帮你分辨什么是"真战略"、什么是"伪战略"，然后用最简单的结构写出一个好战略。适合CEO、创业者、任何需要"把模糊的方向变成清晰行动"的人。

## 核心框架

### 战略内核 (The Kernel of Strategy)

一个好战略只需要三个部分：

| 组成部分 | 核心问题 | 说明 |
|---------|---------|------|
| **诊断 (Diagnosis)** | 我们面临的核心挑战到底是什么？ | 把复杂局面简化为一个清晰的问题定义。不是列出所有问题，而是找到那个最关键的。 |
| **指导方针 (Guiding Policy)** | 我们用什么总体方法来应对这个挑战？ | 一个大方向，说清楚"怎么打"，但不是具体步骤。它排除了很多可能的行动，起到"聚焦"作用。 |
| **连贯行动 (Coherent Actions)** | 具体做哪几件事，而且这些事之间要相互配合？ | 不是一堆零散的To-do，而是一组互相增强的行动。关键词："连贯"。 |

### 坏战略的四个标志

| 标志 | 表现 | 例子 |
|------|------|------|
| **空话连篇 (Fluff)** | 用高大上的词语伪装成战略 | "我们致力于以客户为中心的协同创新" |
| **不直面挑战 (Failure to face the challenge)** | 没有清晰定义核心问题 | 战略文档里没有"问题"只有"目标" |
| **把目标当战略 (Mistaking goals for strategy)** | 只说想去哪里，不说怎么去 | "我们的战略是成为行业第一" |
| **坏的战略目标 (Bad strategic objectives)** | 目标太多、互相矛盾、不可执行 | 列了 20 个"战略重点" |

### 战略杠杆 (Sources of Power)

好战略的力量来自于找到并利用"杠杆点"：

- **聚焦 (Focus)**：把资源集中在最关键的点上，不撒胡椒面
- **优势 (Advantage)**：利用别人没有的能力或位置
- **动态变化 (Dynamics)**：利用行业/技术变化带来的窗口期
- **惯性 (Inertia)**：利用对手因组织惯性而无法快速反应的弱点
- **链条效应 (Chain-link)**：系统中最弱的环节决定整体表现

## 怎么用（操作指南）

### 场景 A：制定公司/业务战略

输入：用户需要为公司或业务制定战略方向

执行步骤：
1. **诊断**：帮用户识别核心挑战。问"如果只能解决一个问题，是什么？"——不是列出所有问题，而是找到那个解开后其他问题迎刃而解的关键问题
2. **指导方针**：基于诊断，提出一个总体应对方法。这个方法必须有取舍——它明确说"做什么"的同时暗示了"不做什么"
3. **连贯行动**：列出 3-5 个具体行动，检查它们之间是否互相配合增强（而不是各自独立甚至矛盾）
4. **坏战略检查**：用四个坏战略标志逆向检验——这个战略有没有空话？有没有回避真正的挑战？是不是把目标当策略了？
5. 输出战略内核文档

### 场景 B：评估/诊断一个现有战略

输入：用户有一份战略文档或战略描述，想知道它好不好

执行步骤：
1. 用坏战略四标志逐条检查——有没有空话？有没有回避挑战？目标当战略？目标太多互相矛盾？
2. 检查战略内核是否完整——有诊断吗？有指导方针吗？行动连贯吗？
3. 如果缺失某个部分，指出来，并帮用户补上
4. 检查"聚焦度"——资源是否集中在最关键的杠杆点上？
5. 给出诊断报告：好战略/坏战略/有框架但需要修正

### 场景 C：把一个模糊的方向变成清晰的战略

输入：用户有一个大致的想法/方向，但说不清楚"战略到底是什么"

执行步骤：
1. 帮用户区分"愿景/目标"和"战略"——"成为第一"是目标不是战略
2. 引导用户做诊断：你面临的最大挑战是什么？为什么其他人没有解决这个问题？
3. 从挑战出发推导指导方针：应对这个挑战，你的总体打法是什么？
4. 把指导方针分解为连贯行动：具体做哪 3-5 件事？这些事之间怎么配合？
5. 反向检验：如果这些行动做成了，是否真的回应了最初的诊断？

## 判断标准

- [ ] **诊断清晰** — 能用一两句话说清核心挑战，不是一堆问题的罗列
- [ ] **指导方针有取舍** — 明确了"不做什么"，不是面面俱到
- [ ] **行动连贯** — 几个行动之间互相配合增强，不是独立的任务清单
- [ ] **没有坏战略特征** — 通过四标志检查，没有空话、没有回避挑战、目标不当战略、目标不矛盾
- [ ] **有杠杆** — 战略利用了某种不对称优势或时机窗口
- [ ] **简洁有力** — 战略内核用一页纸就能说清楚

## 常见误区

1. **把愿景当战略**。"成为行业第一""打造世界级平台"——这是愿景不是战略。战略要回答"怎么做到"。
2. **不做诊断就开药方**。很多人直接跳到"我们应该做XXX"，但没有先搞清楚"我们面临的核心问题到底是什么"。没有诊断的战略像没有诊断的处方。
3. **行动互相矛盾还不自知**。"同时追求极致品质和极低价格""既要深耕本地又要全球扩张"——行动之间的矛盾会互相抵消。
4. **战略重点太多等于没有重点**。如果你有 15 个"战略重点"，你就没有战略重点。好战略意味着聚焦。
5. **战略做完锁抽屉**。战略内核不是年度仪式产出的文档，它应该是每天决策的指南针。

## 组合使用

- **好战略坏战略 + 竞争战略**：Porter 的五力分析为 Rumelt 的"诊断"环节提供工具。先用五力看清行业格局，再用战略内核把洞察转化为行动。
- **好战略坏战略 + 战略选择级联**：Rumelt 的框架更适合"定性判断好不好"，Lafley 的框架更适合"结构化选择怎么做"。可以先用 Rumelt 检查战略质量，再用 Lafley 细化选择。
- **好战略坏战略 + 从0到1**：Rumelt 帮你把 Thiel 的逆向思考落地。Thiel 问"什么是你相信但别人不信的？"，Rumelt 帮你把这个信念变成诊断-方针-行动。

## Agent 指令

### 信息收集
在执行前需要知道：
- 用户是在制定战略、评估战略、还是想把模糊想法变清晰？
- 涉及的业务/公司是什么？
- 目前面临的核心困境或挑战是什么？（如果用户说不清，这本身就是诊断环节的工作）
- 有没有现有的战略文档或方向描述？

先确认任务类型（制定/评估/澄清），再逐步深入。

### 输出格式

**战略内核文档**输出：
```
📋 [公司/业务名] 战略内核

🔍 诊断：
[一段话，清晰定义核心挑战]

🧭 指导方针：
[一段话，说明总体应对方法和关键取舍]

🎯 连贯行动：
1. [行动1] — [为什么做这个 + 和其他行动的配合关系]
2. [行动2] — [……]
3. [行动3] — [……]

⚡ 杠杆点：[这个战略利用了什么不对称优势/时机]

✅ 质量检查：
- 空话：无 ✓
- 直面挑战：是 ✓
- 目标≠战略：是 ✓
- 行动连贯：是 ✓
```

**战略诊断报告**输出：
```
📊 [战略名/公司名] 战略质量诊断

🔴/🟡/🟢 总体判断：[好战略/需修正/坏战略]

坏战略检查：
- 空话连篇：[有/无 + 具体指出]
- 回避挑战：[有/无 + 具体指出]
- 目标当战略：[有/无 + 具体指出]
- 目标矛盾：[有/无 + 具体指出]

战略内核完整性：
- 诊断：[有/缺失/模糊]
- 指导方针：[有/缺失/太泛]
- 连贯行动：[有/缺失/不连贯]

💡 修正建议：[具体怎么改]
```

### 注意事项
- Rumelt 的框架本质是"减法"——帮用户砍掉不重要的，聚焦真正关键的。执行时不要反过来加东西
- 诊断环节可能需要挑战用户的假设——"你确定这是核心问题吗？"——要敢于追问
- 如果用户的"战略"其实是一堆目标，不要直接否定，而是温和引导："这些是很好的目标，现在我们来看看实现它们的战略是什么"
- 语气要像一个资深战略顾问：简洁、犀利、不废话，但也不尖刻
