# 差距销售 (Gap Selling)
> 来源：《Gap Selling》— Keenan（2019）

## 一句话定位

销售的本质不是卖产品，而是帮客户看清"现状"和"理想状态"之间的差距——差距越大，成交越容易。适合需求挖掘、价值呈现、客户诊断场景。

## 核心框架

### 差距模型 (The Gap)

```
[现状] ———差距——→ [期望状态]
   ↓                    ↓
  痛苦                  价值
   ↓                    ↓
  改变的紧迫感          解决方案
```

| 要素 | 定义 | 关键问题 |
|------|------|---------|
| **现状 (Current State)** | 客户今天的处境——环境、问题、影响 | 你现在的情况是什么样的？这给你带来了什么麻烦？ |
| **期望状态 (Future State)** | 客户想要到达的地方 | 如果问题解决了，理想状态是什么？ |
| **差距 (The Gap)** | 现状和期望之间的距离 | 现在和理想之间差多远？这个差距让你损失了什么？ |

### 关键原则

- **没有差距就没有销售**。如果客户觉得现状还行，没有差距，你什么都卖不出去
- **差距越大，紧迫感越强**。你的工作不是推销产品，是帮客户看清差距有多大
- **客户买的不是产品，是"从现状到期望状态的桥"**。如果你的产品不能缩小差距，就不应该卖
- **先诊断，再开方**。不了解客户的现状和期望就推产品，跟医生不做检查就开药一样

### 现状三层挖掘

| 层级 | 内容 | 问什么 |
|------|------|--------|
| **环境层 (Environment)** | 客户的业务背景、流程、工具、团队 | "你们目前怎么做XX？用什么工具？团队怎么分工？" |
| **问题层 (Problems)** | 环境中存在的具体问题和困难 | "这个流程中最大的瓶颈是什么？哪里经常出问题？" |
| **影响层 (Impact)** | 问题对业务的实际影响（收入、成本、效率、士气） | "这个问题每个月让你损失多少？对团队士气有什么影响？" |

**关键**：必须挖到影响层。只知道"有问题"不够，要知道"问题造成了多大的伤害"。影响层是差距的量化证据。

## 怎么用（操作指南）

### 场景 A：做一次高质量的客户需求诊断

输入：用户要和一个潜在客户做需求沟通

执行步骤：
1. 了解用户的产品能解决哪类问题
2. 设计"现状"挖掘问题——从环境到问题到影响，层层深入
3. 设计"期望状态"问题——客户的理想情况是什么？
4. 预设差距量化方法——"这个差距在金额/时间/效率上意味着什么？"
5. 设计"差距确认"环节——把差距总结出来让客户确认
6. 输出诊断对话指南

### 场景 B：写一份有说服力的销售提案

输入：用户需要写销售提案/方案书

执行步骤：
1. 帮用户梳理客户的现状（环境+问题+影响）
2. 明确客户的期望状态
3. 量化差距——这是提案中最有冲击力的部分
4. 展示方案如何弥合差距——不是列功能，而是"功能A解决问题X，缩小差距Y"
5. 提案结构：现状→差距→期望→我们的方案→投资回报
6. 输出提案框架或完整提案

### 场景 C：客户说"不需要"或"以后再说"——重新激活

输入：用户的客户表示暂时不需要或想等等

执行步骤：
1. 诊断原因——通常是因为客户没感受到差距够大
2. 回到影响层——帮用户设计问题，让客户看到"不行动的代价"
3. 量化"等待成本"——每多等一个月/季度损失多少？
4. 设计"温和挑战"话术——不是推销，是"帮你算一笔账"
5. 如果确实差距不大，坦然接受——差距不大说明现在不是对的时机
6. 输出重新激活方案

## 判断标准

- [ ] **现状挖到影响层** — 不只是知道"有问题"，还知道"问题造成了什么损失"
- [ ] **期望状态具体** — 不是"变得更好"，而是"XX指标从A提升到B"
- [ ] **差距可量化** — 差距有数字、有比较、有时间维度
- [ ] **方案对应差距** — 产品功能不是随意展示的，每个功能对应一个具体差距
- [ ] **客户确认了差距** — 差距是客户认同的，不是你单方面认为的

---

## 常见误区

1. **一上来就展示产品功能**。客户不关心你有什么功能，关心的是"你能不能缩小我的差距"。先诊断差距，再展示方案。
2. **只挖到问题层就停了**。"流程慢"是问题，但"流程慢导致每月损失 50 万"才是差距。不挖到影响层，客户不会有紧迫感。
3. **帮客户编造期望状态**。期望状态必须是客户自己表达的，不是你替他想的。你替他编的"理想"他不认账。
4. **差距不大也硬卖**。如果差距确实不大，这不是你的客户（现在不是）。硬卖只会毁掉信任。好的销售知道什么时候不卖。
5. **混淆"差距"和"产品差距"**。差距是客户业务层面的，不是"你没有用我的产品"。把"不用我产品"说成差距，是自说自话。

## 组合使用

- **差距销售 + SPIN**：SPIN 的四类问题可以完美服务差距模型。S 问题挖环境层，P 问题挖问题层，I 问题挖影响层（放大差距），N 问题描绘期望状态。
- **差距销售 + 挑战者销售**：挑战者的"重构"可以帮客户重新定义差距——"你以为差距在效率，其实差距在战略方向"。差距销售提供诊断框架，挑战者提供认知升级。
- **差距销售 + 全新销售**：Pink 的"清晰度"（Clarity）原则——帮客户看清问题——和差距销售的核心理念完全一致。差距销售提供具体方法，Pink 提供底层哲学。

## Agent 指令

### 信息收集
在执行前需要知道：
- 用户的产品/服务是什么？能解决什么类型的问题？
- 目标客户是谁？
- 用户对客户现状了解多少？（如果已做过初步沟通）
- 销售阶段——是第一次接触？已经沟通过但没推进？还是要写提案？
- 客户有没有表达过什么痛点或期望？

先问产品和客户基本信息，通过对话逐步还原差距全貌。

### 输出格式

**差距诊断指南**输出：
```
🔍 [客户名] 差距诊断

📊 现状挖掘问题：
环境层：
1. [问题]
2. [问题]
问题层：
1. [问题]
2. [问题]
影响层：
1. [问题]——量化方向：[收入/成本/效率/风险]
2. [问题]——量化方向：[……]

🎯 期望状态引导：
1. "如果XX问题完全解决，理想状态是什么？"
2. "你希望XX指标达到什么水平？"

📏 差距量化：
[现状指标] → [期望指标] = [差距大小]
换算为：每月/每年损失 [金额/时间/效率]

🌉 方案对应：
- 差距1 → [产品功能/服务A]
- 差距2 → [产品功能/服务B]
```

**销售提案框架**输出：
```
📋 [客户名] 销售提案

1. 现状摘要：[客户当前处境，用客户自己的话]
2. 核心差距：[量化的差距，有数据支撑]
3. 期望状态：[客户确认的理想结果]
4. 解决方案：[每个功能对应一个差距，而不是功能清单]
5. 投资回报：[缩小差距带来的价值 vs 方案成本]
6. 下一步：[具体行动计划]
```

### 注意事项
- 差距销售是"诊断型"方法——像医生一样先诊断再开方
- 量化差距是最有价值的步骤，帮用户把模糊的痛苦变成具体的数字
- 如果客户的差距确实不大，要敢于告诉用户"这个客户现在可能不是优先级"
- 语气要像一个帮你理解客户的教练，不是背公式
