# 狂热开发 (Fanatical Prospecting)
> 来源：《Fanatical Prospecting》— Jeb Blount（2015）

## 一句话定位

销售管道是销售的命脉——没有足够的潜在客户，再好的成交技巧也没用。这本书教你用纪律和系统保持管道永远充满。适合 BD、SDR、任何需要持续获客的销售岗位。

## 核心框架

### 30 天管道法则 (30-Day Rule)

你今天做的开发工作，决定的是 30-90 天后的业绩。

- 如果今天停止开发，30 天后管道就开始干涸
- 销售业绩波动的根本原因：开发活动不持续（忙的时候不开发→空了才慌→开发一阵又忙了→又停了）
- **解法**：每天固定时间做开发，无论多忙都不停。开发是第一优先级，不是"有空才做"

### 替代法则 (Law of Replacement)

管道里的客户始终在流出（成交了、丢了、不回复了）。要保持管道水位，新客户流入速度必须大于流出速度。

公式：**管道水位 = 新开发速度 - 流出速度**

如果你需要每月成交 10 单，成交率是 20%，那你每月需要进入管道的新机会 = 50 个。往回推就知道每天需要多少开发动作。

### 黄金时间法则 (Golden Hours)

- **黄金时间**：工作日中最适合联系客户的时段（通常是上午9-11点、下午2-4点）
- **铁律**：黄金时间只做一件事——联系客户。不开会、不写方案、不回内部邮件
- 非黄金时间做准备工作（研究客户、更新CRM、写邮件模板）

### 多渠道开发 (Multi-Channel Prospecting)

不要只靠一种方式开发客户：

| 渠道 | 特点 | 最佳用法 |
|------|------|---------|
| **电话** | 最快、最直接、拒绝最多 | 适合已有线索的跟进 |
| **邮件** | 可规模化、打开率低 | 适合第一次接触和预热 |
| **社交媒体 (LinkedIn)** | 建立长期关系、预热 | 适合高层目标客户 |
| **面对面** | 效果最好但效率最低 | 适合重要客户和活动场合 |
| **推荐** | 成交率最高 | 适合任何时候——但不能只靠推荐 |
| **短信** | 高打开率、侵入性强 | 适合已有一定关系的跟进 |

**组合使用**：最有效的开发是多渠道组合——先 LinkedIn 互动预热，再邮件介绍，再电话跟进。

### 开发区块 (Prospecting Blocks)

把开发活动切成固定的时间块：

- **高强度区块**（1-2 小时）：集中打电话、发邮件，追求数量
- **准备区块**（30 分钟）：研究目标客户、定制话术
- **社交区块**（30 分钟）：LinkedIn 互动、内容评论

每天至少一个高强度区块。不能被打断。

## 怎么用（操作指南）

### 场景 A：建立个人/团队的开发纪律

输入：用户的销售管道经常断流，业绩忽高忽低

执行步骤：
1. 计算管道数学——目标业绩 ÷ 平均客单价 = 需要成交数 ÷ 成交率 = 需要管道机会数
2. 把管道机会数分解到每天——每天需要多少新联系/新对话
3. 设计每日开发区块——固定时间、固定动作、不可取消
4. 确定黄金时间——根据用户的行业和客户习惯
5. 选择多渠道组合——主要渠道 + 辅助渠道
6. 设计追踪指标——每天做了多少、管道里有多少、推进了多少
7. 输出每日开发作战计划

### 场景 B：设计冷启动的开发策略

输入：用户刚开始一个新业务/新市场，需要从零开始开发客户

执行步骤：
1. 明确目标客户画像——谁是你的理想客户？
2. 建立目标客户清单——从哪里找到这些人？（行业名录、LinkedIn、活动、推荐）
3. 设计多渠道触达顺序——先怎么接触、再怎么跟进
4. 写开发话术/邮件模板——简短、有价值、有行动号召
5. 设计"7 次触达"节奏——研究表明平均需要 7 次以上接触才能获得回应
6. 设定第一个 30 天的开发目标
7. 输出冷启动开发方案

### 场景 C：写开发邮件/电话话术

输入：用户需要具体的开发话术或邮件模板

执行步骤：
1. 了解用户的产品价值和目标客户
2. 设计邮件：短（5 句以内）、有价值（不是推销而是"对你可能有用"）、有明确 CTA
3. 设计电话开场：10 秒说明你是谁、为什么打电话、对客户有什么价值
4. 设计跟进节奏：第 1/3/5/7/14/30 天各做什么
5. 为不同回应准备应对——不回复怎么办、说不需要怎么办、说以后再说怎么办
6. 输出话术和邮件模板套装

## 判断标准

- [ ] **有管道数学** — 清楚知道每天需要多少开发动作才能达成目标
- [ ] **有固定时间** — 开发区块固定在日程里，不是"有空才做"
- [ ] **多渠道组合** — 不是只靠一种方式，至少有 2-3 种渠道配合
- [ ] **有跟进节奏** — 不是联系一次就放弃，有系统的多次触达计划
- [ ] **追踪了活动量** — 每天做了多少、结果如何有记录

## 常见误区

1. **等线索自己来**。"我们有市场部投放""客户会自己找过来"——坐等 inbound 是最大的销售管道杀手。主动开发永远是基本功。
2. **忙的时候停开发**。在跟进很多客户的时候觉得"不用再开发了"——30 天后就会后悔。开发是每天的事，不是管道空了才做。
3. **一次被拒就放弃**。大部分成交需要 5-12 次接触。第一次不回邮件很正常。有系统的跟进节奏才是关键。
4. **只用一种渠道**。只打电话或只发邮件效果都有限。组合使用 2-3 种渠道，总接触率会高得多。
5. **花太多时间在"准备"上**。研究客户很重要，但不能变成"一直在准备、从不动手"。开发是实战活动，需要大量执行。

## 组合使用

- **狂热开发 + SPIN**：Prospecting 负责约到客户，SPIN 负责约到之后怎么挖掘需求。一个解决"见谁"，一个解决"怎么谈"。
- **狂热开发 + 挑战者销售**：开发邮件/电话中可以使用挑战者的"教学"思维——不是推销，而是"分享一个对你有价值的洞察"。让开发信息本身有价值。
- **狂热开发 + 差距销售**：差距销售帮你确定"什么客户有大差距"——这些是你应该优先开发的目标。用差距思维筛选目标，用 Prospecting 方法系统触达。

## Agent 指令

### 信息收集
在执行前需要知道：
- 用户的角色——自己做销售、管理销售团队、还是创业者？
- 卖什么产品/服务？大约客单价多少？
- 目标客户是谁？（行业、规模、职位）
- 当前的开发方式——主要靠什么获客？
- 当前管道情况——有多少活跃机会？够不够？
- 业绩目标——每月/季度需要成交多少？

先问角色和管道情况，再推算数学和设计方案。

### 输出格式

**每日开发作战计划**输出：
```
📊 管道数学：
- 月度目标：[成交XX单 / XX万元]
- 平均成交率：[XX%]
- 需要管道机会：[XX个/月]
- 每日开发目标：[XX个新联系]

⏰ 每日时间安排：
- [时段]：🔴 高强度开发区块（电话+邮件）
- [时段]：🟡 准备区块（研究客户）
- [时段]：🟢 社交区块（LinkedIn）

📱 多渠道节奏（新目标客户7天计划）：
- Day 1：[渠道+动作]
- Day 2：[渠道+动作]
- Day 3：[渠道+动作]
- Day 5：[渠道+动作]
- Day 7：[渠道+动作]

📈 每日追踪：
- 打了多少电话
- 发了多少邮件
- 获得多少回应
- 推进了多少新机会
```

### 注意事项
- 管道数学必须基于用户的真实数字——不要瞎猜成交率，帮用户回顾历史数据
- 开发计划要考虑用户的实际时间——如果还有很多内部会议，要先帮用户"抢"出开发时间
- 话术和邮件模板要短——越短越好。没人读长邮件
- 不要把开发浪漫化——这是一个需要纪律和重复的活动，魅力不重要，执行力才重要
