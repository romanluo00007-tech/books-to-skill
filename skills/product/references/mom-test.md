# 妈妈测试 (The Mom Test)
> 来源：《The Mom Test》— Rob Fitzpatrick（2013）

## 一句话定位

用户访谈中最大的坑是"所有人都说好但没人买"——因为你问错了问题。这本书教你怎么从用户嘴里获取真相而不是礼貌的谎言。适合用户调研、需求验证、创业早期。

## 核心框架

### 妈妈测试三原则

为什么叫"妈妈测试"？因为如果你的问题连妈妈都会出于爱给你假信息，说明你问错了。

| 原则 | 做什么 | 不做什么 |
|------|--------|---------|
| **聊他们的生活，不是你的想法** | "你现在怎么处理XX问题？" | "你觉得我这个产品怎么样？" |
| **问过去的事实，不是未来的假设** | "上次遇到这个问题你怎么做的？" | "如果有这样一个功能你会用吗？" |
| **少说多听** | 让对方说 80%，你说 20% | 滔滔不绝介绍你的想法然后问"你觉得呢" |

### 好问题 vs 坏问题

| 坏问题（得到假信息） | 好问题（得到真信息） | 为什么 |
|---------------------|---------------------|--------|
| "你觉得这个想法怎么样？" | "你现在怎么解决这个问题？" | 观点不值钱，行为才值钱 |
| "你会买这个产品吗？" | "你上次为解决这个问题花了多少钱？" | 未来承诺是假的，过去行为是真的 |
| "你想要XX功能吗？" | "上次没有XX功能时你怎么办的？" | 需求要从困境中推断，不能直接问 |
| "你觉得这值多少钱？" | "你现在用的方案花了多少钱/时间？" | 让用户定价是假数据 |

### 承诺与推进 (Commitment & Advancement)

对话结束后，怎么判断对方是真感兴趣还是客气？看承诺：

| 承诺类型 | 含金量 | 例子 |
|---------|--------|------|
| **时间承诺** | 中 | 愿意再聊一次、参加测试 |
| **声誉承诺** | 高 | 愿意介绍给同事/朋友 |
| **金钱承诺** | 最高 | 愿意预付、愿意签LOI |

**关键**：如果对方只说"好主意！"但不愿意花时间、花钱、或介绍别人——这是假阳性。

### 三种坏数据

| 坏数据类型 | 表现 | 怎么识别 |
|-----------|------|---------|
| **恭维** | "这个想法太棒了！" | 好听但没有行动承诺 |
| **空洞的假设** | "我绝对会用的！" | 对未来的承诺，没有过去行为支撑 |
| **闲聊** | 聊了很久但没有有价值的信息 | 聊完后你没有新学到什么 |

## 怎么用（操作指南）

### 场景 A：为新产品做用户调研

输入：用户想验证一个产品想法，需要做用户访谈

执行步骤：
1. 明确要验证的核心假设——不是"用户喜不喜欢"，而是"用户有没有这个问题、问题严不严重"
2. 设计访谈问题——全部是关于过去行为和事实的问题，不问观点和假设
3. 设计承诺请求——访谈结束时要什么承诺来验证真伪
4. 帮用户找合适的访谈对象——不是朋友和家人，是真正的目标用户
5. 写出完整的访谈指南
6. 提醒用户：一次访谈不要超过 3 个核心问题

### 场景 B：分析已有的用户反馈是否可靠

输入：用户做了一些用户访谈或收到了反馈，想判断信息质量

执行步骤：
1. 检查问题质量——问的是观点还是事实？是未来假设还是过去行为？
2. 检查坏数据——有没有恭维、空洞假设、闲聊？
3. 检查承诺——对方有没有给出时间/声誉/金钱承诺？
4. 筛选出可靠信息——哪些是基于过去行为的真实数据？
5. 给出改进建议——下一轮访谈应该怎么问？
6. 输出反馈质量诊断报告

### 场景 C：为用户设计访谈指南模板

输入：用户需要一套可以反复使用的访谈方法

执行步骤：
1. 了解用户的产品方向和目标客户
2. 设计通用开场——怎么让对方放松且聚焦在他们的经历上
3. 设计 5-7 个核心问题——全部符合妈妈测试原则
4. 设计跟进问题——对方说了有趣的事情后怎么深挖
5. 设计收尾——怎么请求承诺，怎么请求介绍更多人
6. 附上"不要做"清单——常见的错误提醒
7. 输出访谈指南模板

## 判断标准

- [ ] **问的是过去行为** — 不是"你会不会""你觉得""你想不想"
- [ ] **没有推销** — 访谈中没有介绍产品或暗示正确答案
- [ ] **有事实支撑** — 每个发现都有具体的过去行为作为证据
- [ ] **有承诺验证** — 不是靠"听起来对方很感兴趣"判断，而是有具体承诺
- [ ] **学到了意料之外的东西** — 如果访谈只是确认了你已有的想法，说明问题问得不够好

---

## 常见误区

1. **带着方案去问**。"我们做了个XX产品，你觉得怎么样？"——这是在推销，不是在调研。对方只会客气地说"挺好的"。
2. **问未来意向**。"如果有这个功能你会用吗？"——100% 的人都说会，但真到了只有 5% 会。问过去做了什么，不问将来会不会。
3. **问太广的问题**。"你工作中最大的困难是什么？"——太大了，对方会给一个社交性答案。要具体："上周有没有遇到XX情况？当时怎么处理的？"
4. **把恭维当验证**。别人说"这个主意真好"不等于验证通过。只有承诺（时间、声誉、金钱）才是真信号。
5. **只访谈不记录**。每次访谈后立刻记录关键发现。过两天你会忘记 80%，而且记忆会被你的期望扭曲。

## 组合使用

- **妈妈测试 + 精益创业**：精益的"构建-测量-学习"中，妈妈测试是"测量"和"学习"的核心方法——帮你从用户那里获取真实信息。
- **妈妈测试 + JTBD**：JTBD 告诉你要研究什么（用户的 Job、情境、进展），妈妈测试告诉你怎么正确地研究（问过去行为、不问观点）。
- **妈妈测试 + 启示录**：Cagan 强调用户访谈在产品发现中的核心作用，妈妈测试提供具体的访谈方法论。
- **妈妈测试 + 设计冲刺**：Sprint 的 Day 5 用户测试可以借鉴妈妈测试的原则——观察行为、不引导答案。

## Agent 指令

### 信息收集
在执行前需要知道：
- 用户想验证什么？（产品想法、需求假设、市场方向）
- 目标客户是谁？
- 有没有已经做过的访谈？结果如何？
- 访谈对象从哪里找？
- 核心假设是什么——"如果这不成立，整个方向就错了"的那个假设？

先问想验证什么和目标用户。

### 输出格式

**访谈指南**输出：
```
🎙️ [产品/方向] 用户访谈指南

🎯 本次要验证的假设：[明确写出]

👋 开场（2分钟）：
"[自然的开场白，让对方聊自己的经历而不是你的产品]"

❓ 核心问题（按顺序问）：
1. [关于过去行为的问题] — 想了解：[什么信息]
2. [关于过去行为的问题] — 想了解：[什么信息]
3. [关于过去行为的问题] — 想了解：[什么信息]

🔍 跟进追问（当对方说了有趣的事）：
- "能具体说说那次的情况吗？"
- "那后来怎么处理的？"
- "这个问题多久出现一次？"

🤝 收尾 + 承诺请求：
- "有没有其他遇到同样问题的朋友可以介绍给我聊聊？"（声誉承诺）
- "我们下周可以再聊一次吗？"（时间承诺）
- "如果我们做出原型，你愿意试用并给反馈吗？"（时间承诺）

🚫 不要做：
- 不要介绍你的产品想法
- 不要问"你觉得XXX怎么样"
- 不要问"你会不会用XXX"
- 不要接受恭维作为验证
```

### 注意事项
- 妈妈测试的核心是"让用户说真话"——这需要你不暴露自己的期望
- 每次访谈聚焦 3 个核心问题就够了——太多问题对方会疲劳
- 访谈后立刻记录——不要相信你的记忆
- 5 次访谈就能看到模式——如果 5 个人都说同一个事情，那就是真需求
- 这个方法适合 B2C 和 B2B，但 B2B 的承诺信号更容易判断（愿不愿意付费试用）
