# Skill-to-Showcase

把任何领域的方法论 skill 变成一个交互式展示网页。

这个网页的定位是 **skill 的营销页面**：给 OpenClaw 用户看，让他们了解这个 skill 有什么、怎么用，然后一键安装。

---

## 网页包含什么

生成的网页有四个 Tab：

### 首页（营销页）
- Skill 名称 + Slogan
- 一键安装命令（可复制）
- 本 Skill 包含：卖点提炼（3 条）
- 使用场景：3 个 persona 卡片
- 使用方法：3 步可视化流程（学习→自动调用→指定框架）

### 浏览（知识库）
- 搜索框
- 按金字塔支柱分组展示所有方法论
- 每个卡片展示：书名、作者、一句话定位、标签、书封
- 点击卡片弹出详情 Modal：核心框架（步骤）、常见误区、组合使用、小测试

### 诊断（场景匹配）
- 按金字塔支柱分组的场景列表，左侧流程线可视化
- 选择一个场景 → 推荐首选+补充方法论 + 推荐理由 + prompt 示例（可复制）

### 测试（知识检验）
- N 道单选题（等于方法论数量），随机顺序
- 九宫格进度指示，红绿反馈
- 答错可跳转方法论详情
- 结果页：评级 + 建议复习的方法论

## 生成流程

### 第 0 步：确认输入

需要用户提供以下之一：
- **一个已有的领域 skill**（SKILL.md + references/）→ 从中提取数据
- **核心书库中的一个领域** → 先生成 skill，再生成网页
- **用户自定义的方法论列表** → 按规范整理数据

### 第 1 步：准备数据

阅读数据规范：
```
data-spec.md
```

按规范生成四组数据：
1. **siteConfig**：网站配置（名称、slogan、色调、安装命令、personas、usageSteps）
2. **PYRAMID**：金字塔支柱（2-4 个，有逻辑递进）
3. **FW**：方法论列表（每个含完整数据：steps、traps、combos、quiz）
4. **SCENES**：场景列表（每个支柱 3-5 个，含推荐理由和 prompt 示例）

### 第 2 步：查找书籍封面

对每本书，尝试从 Open Library 获取封面 ID：
```
https://openlibrary.org/search.json?title=BOOK_TITLE&author=AUTHOR
```
取 `cover_i` 字段。找不到就设为 `null`。

### 第 3 步：组装网页

将数据填入 App.jsx 模板。模板在：
```
templates/app-template.jsx
```

模板中有明确的数据占位区域（`// === DATA START ===` 到 `// === DATA END ===`），只需替换这部分。

模板的 UI 结构、交互逻辑、样式都不需要修改。

### 第 4 步：输出

输出一个完整的 Vite + React 项目：
```
[skill-name]-showcase/
├── index.html
├── package.json
├── vite.config.js
└── src/
    ├── main.jsx
    └── App.jsx
```

用户可以直接 `npm install && npm run dev` 本地预览，或部署到 Vercel。

## 关键质量标准

### 金字塔设计
- 支柱之间必须有逻辑递进（先→后），不是平行分类
- 每个方法论只属于一个支柱
- 支柱命名要简洁有力（2-4 字），用动词短语

### 方法论数据
- `one`（一句话定位）：必须具体、有场景感，不能是"帮你做XX"这种废话
- `steps`：每步的 `d` 要有具体操作指引，不是泛泛而谈
- `traps`：要能区分专家和新手——专家知道什么时候不该用
- `combos`：说明具体怎么配合、什么场景用，不只是"可以一起用"
- `quiz`：考理解不考记忆，错误选项要代表常见误解

### 场景设计
- `t`（用户问题）：用第一人称口语，像真人说话
- `prompts`：用户可以直接复制给 agent 的完整 prompt，不是半句话
- 每个支柱下的场景要覆盖该阶段的常见需求，不重复

### 视觉统一
- `accentColor` 不要和营销版重复（营销是金色 #f0c040）
- 建议各领域色调：
  - 战略：#6366f1（靛蓝）
  - 销售：#22c55e（绿色）
  - 管理：#f97316（橙色）
  - 产品：#06b6d4（青色）
  - 领导力：#a855f7（紫色）
  - 个人成长：#14b8a6（薄荷绿）
  - 沟通：#ec4899（粉色）
  - 思维决策：#64748b（灰蓝）

## 从已有 skill 提取数据的方法

如果输入是一个已完成的领域 skill（如 marketing/），按以下方式提取：

1. **FW**：从每个 `references/*.md` 中提取
   - `name/en`：文件标题
   - `book/author/year`：来源信息
   - `one`："一句话定位"部分
   - `idea`："核心框架"开头的概述
   - `steps`：核心框架的步骤/维度
   - `traps`："常见误区"部分
   - `combos`："组合使用"部分
   - `quiz`：需要新生成（reference 中没有 quiz 数据）

2. **PYRAMID**：从 SKILL.md 的方法论清单和场景映射表中归纳支柱分组

3. **SCENES**：从 SKILL.md 的"场景→方法论映射"表扩充
   - 每行映射变成 1-2 个场景
   - 补充 `w`（推荐理由）和 `prompts`（prompt 示例）

4. **siteConfig**：根据领域特点生成

## 注意事项

- 网页是给人看的营销页面，不是技术文档。语气要有亲和力
- 每个领域的 persona 和 usageSteps 要定制，不能照抄营销版
- coverId 找不到时就留 null，不要编造
- 生成的 quiz 要先自己验证答案是否正确
- 首页的"本 Skill 包含"要突出这个领域的独特价值，不是通用描述
