# 展示网页数据规范

任何一个领域的 skill，只要按以下规范提供数据，就能自动生成一个交互式学习展示网页。

---

## 数据结构

### 1. 网站配置 (siteConfig)

```json
{
  "skillName": "marketing-playbook",
  "title": "营销知识库 Skill",
  "subtitle": "营销方法论",
  "slogan": "让 AI 开口就是营销顾问",
  "description": "9 本营销必读经典提炼 · 可执行框架",
  "installCmd": "clawhub install marketing-playbook",
  "accentColor": "#f0c040",
  "accentGradient": ["#f0c040", "#e67e22"],
  "bookCount": 9,
  "frameworkCount": 9,
  "sceneCount": 19,
  "quizCount": 9,
  "personas": [
    { "title": "营销人冷启动龙虾", "desc": "一键安装，龙虾秒懂营销", "color": "#e74c3c" },
    { "title": "营销人日常使用", "desc": "定位、文案、转化、留存…随时调取方法论", "color": "#3498db" },
    { "title": "想学习营销经典理论", "desc": "通过实战 prompt 掌握 9 本经典", "color": "#1abc9c" }
  ],
  "usageSteps": [
    { "step": 1, "title": "让 Agent 学习", "examples": ["你去学一下营销方法论"], "result": "升级龙虾大脑", "color": "#f0c040" },
    { "step": 2, "title": "在相关任务中自动调用", "examples": ["帮我分析这条内容为什么火", "帮我写卖点"], "result": "Agent 自动匹配框架", "color": "#e67e22" },
    { "step": 3, "title": "指定具体框架进行分析", "examples": ["用 STEPPS 帮我分析"], "result": "精确调用某一框架", "color": "#e74c3c" }
  ]
}
```

**生成指南：**
- `skillName`：skill 的安装 ID
- `title`/`subtitle`/`slogan`：首页大标题、标签、一句话卖点
- `accentColor`：主色调，贯穿全站。不同领域建议用不同色调区分
- `personas`：3 个使用场景卡片，每项需 `title`、`desc`、`color`。视觉上用彩色圆点+左侧色条，无需 icon/emoji
- `usageSteps`：3 步使用方法，每项需 `step`(1-3)、`title`、`examples`(数组)、`result`、`color`。从"学习"→"自动调用"→"指定框架"，examples 要换成该领域的典型 prompt

---

### 2. 金字塔结构 (PYRAMID)

将该领域的方法论按逻辑链路分组为 2-4 个支柱（阶段/层次/维度），形成一条完整的思维链。

```json
[
  {
    "pillar": "定战略",
    "desc": "我是谁？占什么位置？讲什么故事？",
    "color": "#e74c3c",
    "ids": [0, 5, 3]
  }
]
```

**生成指南：**
- 支柱数量 2-4 个，太多会碎，太少没结构
- 支柱之间要有逻辑递进关系（先做什么→再做什么→然后什么→最后什么）
- `ids` 数组引用 FW 中的方法论 id
- 每个支柱一个主色调（`color`），颜色之间要有区分度。视觉上用数字+左侧色条，无需 emoji
- `desc` 用一句提问式的话，不超过 20 字

**各领域参考结构：**

| 领域 | 支柱建议 |
|------|---------|
| 营销 | 定战略→做表达→促转化→保留存 |
| 战略 | 看清局势→找到方向→设计打法→执行落地 |
| 销售 | 找到客户→理解需求→说服成交→维护关系 |
| 管理 | 定目标→搭团队→促执行→建文化 |
| 产品 | 发现需求→验证方案→打造产品→推向市场 |
| 领导力 | 认知自我→激励他人→建设团队→引领变革 |

### 3. 方法论列表 (FW)

每个方法论是一条完整的数据记录：

```json
{
  "id": 0,
  "name": "定位理论",
  "en": "Positioning",
  "emoji": "📍",
  "book": "Positioning",
  "author": "Ries & Trout",
  "year": 1981,
  "color": "#e74c3c",
  "coverId": 55745,
  "one": "在用户心智中占据一个独特位置",
  "tags": ["品牌定位", "竞争差异化"],
  "idea": "营销的终极战场不是货架，是用户心智。...",
  "steps": [
    { "n": "分析竞争环境", "d": "这个品类里已经有谁？..." },
    { "n": "找到空位", "d": "哪些位置被占了..." }
  ],
  "traps": ["把定位等同于slogan", "定位太宽什么都想是"],
  "combos": [
    { "with": "紫牛", "how": "定位告诉你占据什么位置，紫牛让产品成为证据" }
  ],
  "q": "以下哪个是好的定位声明？",
  "opts": [
    { "t": "我们为所有人提供最好的服务", "c": false, "e": "太宽泛" },
    { "t": "面向独立开发者的10分钟部署后端服务", "c": true, "e": "有明确人群和排他价值" }
  ]
}
```

**各字段说明：**

| 字段 | 必填 | 说明 |
|------|------|------|
| id | ✅ | 从 0 开始的整数，被 PYRAMID.ids 和 SCENES.p/s 引用 |
| name | ✅ | 中文名，3-8 字 |
| en | ✅ | 英文名，用于搜索和显示 |
| emoji | 可选 | 1 个 emoji，无 coverId 时作为书封占位显示。有 coverId 可不填 |
| book | ✅ | 来源书名（简写即可） |
| author | ✅ | 作者名 |
| year | ✅ | 出版年份 |
| color | ✅ | 主色，十六进制，和所属支柱色调一致或相近 |
| coverId | 可选 | Open Library 封面 ID，用于展示书封图片 |
| one | ✅ | 一句话定位，15-25 字 |
| tags | ✅ | 2-3 个标签，标明适用场景 |
| idea | ✅ | 核心理念说明，50-100 字。不是书摘，是"这个理论的底层逻辑是什么" |
| steps | ✅ | 核心框架步骤，每步有 n(名称) 和 d(描述)。4-7 步 |
| traps | ✅ | 常见误区，3-5 条，每条一句话 |
| combos | ✅ | 组合使用，2-3 条，说明和哪个理论配合、怎么配合 |
| q | ✅ | 测试题题干 |
| opts | ✅ | 4 个选项，每个有 t(文字)、c(是否正确)、e(解释)。有且仅有 1 个正确 |

**steps 写法要点：**
- `n`：步骤名称。如果有英文术语，格式为"中文 English"（如"社交货币 Social Currency"）
- `d`：步骤描述。用大白话，给出关键问题或操作要点，50-80 字
- 步骤之间要有逻辑递进，不是平行罗列

**测试题写法要点：**
- 题目考察的是对理论的理解而非记忆
- 正确选项不应该是"显而易见"的，要有真正的辨析价值
- 错误选项要有代表性——代表一种常见的误解
- 解释要简短有力，一句话说清为什么对/错

---

### 4. 场景列表 (SCENES)

每个场景是用户的一句真实问题 → 推荐方法论：

```json
{
  "t": "我的品牌应该怎么定位？",
  "p": 0,
  "s": 5,
  "l": "品牌定位",
  "pillar": 0,
  "w": "定位是一切营销动作的起点...",
  "prompts": [
    "帮我分析XX行业的竞争格局，找到定位机会",
    "我的产品是XX，目标用户是XX，帮我做定位",
    "诊断一下我们现在的品牌定位有什么问题"
  ]
}
```

| 字段 | 必填 | 说明 |
|------|------|------|
| t | ✅ | 用户的原话，用第一人称，口语化 |
| p | ✅ | 首选方法论的 id（引用 FW） |
| s | ✅ | 补充方法论的 id，没有则为 null |
| l | ✅ | 场景标签，2-4 字 |
| pillar | ✅ | 所属支柱的 index（0-3） |
| w | ✅ | 推荐理由，30-60 字 |
| prompts | ✅ | 3 条 prompt 示例，用户可以直接复制给 agent 用。格式为「动作+具体对象」 |

**场景数量建议：**
- 每个支柱 3-5 个场景
- 总计 12-20 个
- 场景要来自用户真实会说的话，不是理论术语

---

## 数据生成流程

给 Agent 的指令：当需要为一个领域的 skill 生成展示网页数据时：

1. **加载该领域的 SKILL.md 和 references/**，理解所有方法论
2. **设计 PYRAMID**：找到 2-4 个支柱，形成逻辑链
3. **生成 FW**：为每个方法论填充完整数据，特别注意 steps 的质量和 quiz 的辨析度
4. **生成 SCENES**：从 SKILL.md 的场景映射表出发，扩充到每个支柱 3-5 个
5. **生成 siteConfig**：品牌信息、色调、personas、usageSteps
6. **组装进模板**：将数据填入 App.jsx 模板

## coverId 查找方法

Open Library 封面 ID 可以通过以下方式获取：

1. 搜索 `https://openlibrary.org/search.json?title=BOOK_TITLE&author=AUTHOR`
2. 从结果中找到 `cover_i` 字段
3. 封面 URL 格式：`https://covers.openlibrary.org/b/id/{coverId}-M.jpg`

如果找不到封面，`coverId` 设为 `null`，网页会自动用 emoji 代替。
